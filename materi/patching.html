Patching & Pengambilan Data via SSH

Panduan praktis: cek IP, update/patch Fedora, akses via SSH, ambil file/database ke CPU

Ringkasan Singkat

Alur umum setelah instalasi OS pada server parkir:

  1. Cek alamat IP & konektivitas
  2. Akses server via SSH (hindari telnet)
  3. Lakukan update/patching (dnf)
  4. Ambil data yang diperlukan ke CPU/komputer admin (scp/rsync/ssh)
  5. Restart layanan & cek log

1) Cek IP & Konektivitas

Beberapa perintah berguna di terminal Fedora:


ip addr show
ip route
ping -c 4 192.168.1.10
nc -vz 192.168.1.10 22

2) Akses Server — SSH

Umumnya akses remote dimulai dengan ssh root@IP:


ssh root@192.168.1.10
ssh -p 2222 root@192.168.1.10

Catatan: jangan gunakan telnet, gunakan SSH terenkripsi. Gunakan key pair untuk login tanpa password interaktif.

3) Patching / Update Fedora


sudo dnf check-update
sudo dnf update -y
sudo reboot
uname -r
sudo systemctl status sshd
sudo journalctl -u your-service-name -n 50

4) Mengambil File / Data ke CPU

Gunakan scp atau rsync:


scp root@192.168.1.10:/var/log/parking/app.log ./app.log
rsync -avz -e "ssh -p 22" root@192.168.1.10:/var/lib/mysql/ /backup/mysql/
ssh root@192.168.1.10 "mysqldump -u dbuser -p'password' parking_db" > parking_db_dump.sql
ssh root@192.168.1.10 "tar czf - /var/log/parking" > parking_logs.tar.gz

5) Contoh Workflow

  1. SSH ke server: ssh root@192.168.1.10
  2. Periksa IP & kondisi: ip addr; free -h; df -h
  3. Ambil snapshot log atau DB: ssh root@192.168.1.10 "tar czf - /var/log/parking" > /home/admin/parking_logs_$(date +%F).tar.gz
  4. Jalankan update: sudo dnf update -y
  5. Restart service: sudo systemctl restart nginx
  6. Periksa log & status: sudo journalctl -u nginx -n 50
  7. Sinkron file konfigurasi via scp atau rsync

6) Keamanan & Praktik Baik

  • Jangan pakai telnet, gunakan SSH terenkripsi.
  • Matikan login root via password (/etc/ssh/sshd_config) → restart sshd.
  • Gunakan key-based auth dan batasi IP.
  • Backup sebelum patching.
  • Update diuji di staging server dulu.

7) Troubleshooting Cepat

  • SSH timeout: cek firewall, sshd, port
  • scp/rsync gagal: cek storage & permission
  • Service gagal restart: cek journalctl -xe
